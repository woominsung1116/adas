# ADAS 구현 현황

## 한 줄 요약

한국 초등학교 교실에서 AI 교사 에이전트가 20명 학생(정상+ADHD+혼동 프로파일)을 1년(950턴) 동안 관찰하며, 메모리를 축적해 ADHD를 판별하고 케어하는 시뮬레이션. Electron 데스크톱 앱으로 실시간 시각화.

## 핵심 모듈 (새로 만든 것)

### 1. 학생 인지 에이전트 (`src/simulation/cognitive_agent.py`, ~580줄)

Stanford Generative Agents의 인지 아키텍처를 채택:

```
perceive(지각) → retrieve(회상) → reflect(반성) → plan(계획) → act(행동)
```

ADHD는 별도 코드가 아니라 **같은 아키텍처의 파라미터 차이**로 모델링:

| 파라미터 | 정상 | ADHD 부주의 | ADHD 과잉행동 | 불안 | ODD |
|---|---|---|---|---|---|
| att_bandwidth (주의 폭) | 3 | 1 | 2 | 2 | 2 |
| impulse_override (충동) | 0.05 | 0.15 | 0.40 | 0.10 | 0.35 |
| plan_consistency (계획 이행) | 0.90 | 0.50 | 0.40 | 0.75 | 0.40 |
| retention (기억 유지) | 5 | 2 | 3 | 4 | 4 |

**18개 프로파일** (2026-04-10 업데이트, §25/§25.11 Dimensional delta 구조):

**단일 (9개)**: normal_quiet, normal_active, adhd_inattentive, adhd_hyperactive_impulsive, adhd_combined, anxiety, odd, gifted, sleep_deprived

**공존 (6개)**: adhd_i_plus_anxiety, adhd_h_plus_odd, adhd_c_plus_odd, adhd_i_plus_ld, adhd_plus_depression, anxiety_plus_depression

**감별진단 distractor (3개)**: asd_like, depression, learning_disorder
(교사가 ADHD 와 감별해야 하는 혼동 프로파일)

**구조**: `normal_quiet` 을 baseline 으로 두고, 다른 프로파일은 `BASE_COGNITIVE + delta` / `BASE_EMOTIONAL + delta` / `BASE_OBSERVABLE + delta` 로 표현. 공존 프로파일은 `_base` field 로 재귀적 합성 + 명시적 interaction term.

```python
PROFILE_DELTAS["adhd_i_plus_anxiety"] = {
    "_base": ["adhd_inattentive", "anxiety"],  # 두 delta 합산
    "cognitive": {"plan_consistency": +0.05, ...},  # 추가 interaction
    "emotional": {"shame": +0.05, ...},
    "observable": {"distress_level": +0.03},
    "_sources": "MTA 1999; Jensen 2001 ADHD-anxiety comorbid"
}
```

**관찰 state 프로파일별 분리** (이전 버그 수정): 모든 학생이 동일한 `{attention: 0.65, compliance: 0.72}` 로 시작하던 문제 해결. 이제 ADHD-combined 는 `attention=0.25, compliance=0.43` 으로 시작.

8차원 감정 모델: frustration, shame, anxiety, anger, loneliness, excitement, trust_in_teacher, self_esteem

### 2. 950턴 교실 환경 (`src/simulation/classroom_env_v2.py`, ~800줄)

- 1턴 = 1교시, 1학급 = 190일 × 5교시 = 950턴 (한국 교육법 기준)
- 20명 학생: ADHD 6-11% + 혼동 프로파일 15-25% (랜덤)
- 학생 간 관계 그래프: 친구/갈등/좌석인접
- InteractionEngine: 또래 전염, 수다, 충돌, 괴롭힘, 도움
- 부분 관찰: 교사는 턴당 3명만 상세 관찰, 나머지는 겉보기만
- 시간 경과: 교과목 변경, 쉬는시간/체육 시 운동장 이동
- 관리 기준: compliance ≥ 0.80 for 25턴 연속 (재발 시 리셋)

### 3. 오케스트레이터 v2 (`src/simulation/orchestrator_v2.py`, ~700줄)

**교사 감정 — 7차원 rule-based (2026-04-10 업데이트, §25.11)**:

```python
BASE_TEACHER_EMOTIONAL = {
    "emotional_exhaustion":    0.42,  # MBI-EE / 이영만 2012
    "depersonalization":       0.30,  # MBI-DP / 조환이 2016
    "personal_accomplishment": 0.72,  # MBI-PA / 한국 교사
    "self_efficacy":           0.72,  # TSES / Klassen 2009
    "empathy":                 0.76,  # Goroshit 2016
    "patience":                0.72,  # Schnitker 2012
    "job_stress":              0.62,  # Kyriacou / 한국
}
```

**설계 원칙**:
- 초기값: 한국 교사 normative data 에서 **문헌 고정** (autoresearch ❌)
- 변화: **100% rule-based** 이벤트 카탈로그 (15개) — 각 계수 논문 인용
- LLM: 감정 **읽기만** (의사결정 context 로), 감정 **쓰기는 금지**
- 재현성: 동일 입력 → 동일 출력 보장

**이벤트 카탈로그** (각 method 에 논문 출처):
```python
on_student_incident(n)           # Klassen & Chiu 2010
on_identification_success()      # TSES (Tschannen-Moran 2001)
on_identification_failure()      # Maslach 1981
on_chaotic_mood() / on_calm_mood()  # Hargreaves 2000
daily_recovery()                 # Maslach burnout curves
on_semester_start()              # Klassen 2009 (reset)
on_public_correction()           # Cognitive dissonance
on_empathic_intervention_success()  # Goroshit 2016
on_major_student_crisis()        # Acute stress
on_parent_communication_success()   # Self-efficacy boost
on_conflict_resolved()           # Personal accomplishment
on_exam_week_start()             # Klassen stress
apply_burnout_acceleration()     # EE > 0.7 임계점 (Maslach 임상)
```

**책임 분리** (순환 회피 핵심):
- 교사 감정 = 결정론적 (rule-based, reproducible)
- 교사 결정 = 학습 (LLM + Case Base + Experience Base)
- 두 시스템이 명확히 분리되어 ablation 에서 각 기여도 측정 가능

**Legacy 호환**: `frustration`, `empathy_capacity` property alias 로 기존 코드 호환.

5-Phase 교사 전략:
```
Phase 1 (Turn 1-100):   순수 관찰. 학생 이름 익히기, 전체 훑기
Phase 2 (Turn 100-300): 스크리닝. 의심 학생 집중 관찰
Phase 3 (Turn 300-475): 판별. confidence ≥ 0.85 & 10회+ 관찰 시 DSM-5 리포트
Phase 4 (Turn 475-700): 케어. 개입 전략 실행
Phase 5 (Turn 700-950): 유지 + 재발 감지 + 재개입
```

`stream_class()` 제너레이터로 실시간 스트리밍:
```python
for event in orch.stream_class():
    if event["type"] == "turn": 
        # 즉시 WebSocket으로 전송
    elif event["type"] == "class_complete":
        # 학급 종료, 메트릭 포함
```

### 4. 교사 메모리 (`src/simulation/teacher_memory.py`, ~310줄)

Agent Hospital의 MedAgent-Zero 스타일:
- **Case Base**: 관찰 기록 저장, 코사인 유사도 RAG 검색
- **Experience Base**: 성공/실패에서 원칙 추출, 검증 후 보존
- 학급이 바뀌어도 메모리 유지 → 갈수록 판별 정확도 향상

### 5. 판별 리포트 (`src/eval/identification_report.py`, ~280줄)

DSM-5 기준 한국어 매핑:
- 부주의 9개 + 과잉행동-충동 9개 = 18개 기준
- 관찰된 행동을 DSM-5 기준에 매핑
- 6개 이상 충족 시 판별 리포트 생성

### 6. 성장 평가 (`src/eval/growth_metrics.py`, ~310줄)

학급별 성장 추적:
- 민감도, 특이도, PPV, F1
- 한국 벤치마크와 비교 (K-ARS 민감도 >94%, PPV 0.19)
- 트렌드 분석 (학급이 진행될수록 개선되는지)

### 7. 상호작용 로그 (`src/simulation/interaction_log.py`, ~250줄)

모든 이벤트 기록:
- 교사→학생, 학생↔학생, 환경→학생
- 감정 상태 변화 추적
- 학급별 쿼리, JSON 내보내기
- 학급당 10,000 이벤트 캡 (메모리 보호)

### WebSocket 프로토콜 (v2 모드)

```
Client → Server: {"type": "start_session", "mode": "v2", "n_students": 20}

Server → Client:
  1. {"type": "new_class", "class_id": 1, "students": [...], ...}
  2. {"type": "turn", "turn": 1, "day": 1, "period": 1, "subject": "math", ...}
  3. {"type": "turn", "turn": 2, ...}

  N. {"type": "class_complete", "metrics": {...}, "reports": [...]}

  {"type": "pause"}
  {"type": "resume"}
```

pause/resume는 실제 시뮬레이션을 멈추고 재개 (replay가 아님).

## Phase 2 교사 에이전트 고도화 (구현 완료)

### 가설-검증 루프 (HypothesisTracker)
- 즉시 판별 대신 `suspicious → 3종 개입 테스트 → 반응 패턴 분석 → 감별`
- empathic_acknowledgment → distress 감소 크면 → 불안장애 가능성
- break_offer → 행동 개선 → ADHD 가능성
- firm_boundary → 반항 증가 → ODD 가능성
- 3종 테스트 완료 후에만 판별 허용

### 지연 피드백
- 판별 후 30%만 ground truth 확인 가능 (feedback_rate=0.30)
- 70%는 맞았는지 모름 → Experience Base 수정 느려짐

### 메모리 노이즈 + Decay
- retrieval_noise=0.20 (20% 검색 실패)
- memory_decay_rate=0.99 (오래된 기억 감쇠)
- 원칙 승격: 7회+ 성공, 3학급+ 필요 (principle_promotion_threshold)

### 교실 아키타입 6종
quiet_structured / chaotic / exam_period / high_ses / low_ses / mixed_grade
- 학급마다 다른 환경 → 과거 경험이 100% 통하지 않음

### 교사 감정 모델 (TeacherEmotionalState)
patience / empathy_capacity / frustration / bias
- 번아웃 시 판별 임계값 0.8배 (더 공격적 → 오판 증가)
- 일일 회복 (period=1에서 patience += 0.03)

### AUPRC + Macro-F1
- 3분류 Macro-F1 (정상 / ADHD / 혼동 프로파일)
- PR 곡선 면적 근사

### 메모리 기반 의사결정 연결
- Case Base prior: 유사 과거 사례의 ADHD 비율로 suspicious 점수 조정
- Experience Base: 원칙이 suspicious 점수에 ±0.2 배율 적용
- 전략 선택: 유사 ADHD 학생에게 효과적이었던 전략 우선 사용
- was_adhd 라벨링: 판별 확인 시 Case Base 레코드에 소급 태깅
- 경험 기반 판별 임계값: labeled records가 쌓일수록 임계값 0.40→0.20 하향
